# Releases

Store user-facing release notes here.

Each release note should live at `docs/releases/vX.Y.Z.md` and answer:

- Summary
- What Changed
- Why It Matters
- Breaking Changes
- Migration
- Links to deeper docs

If there is no upgrade work for users, say `No migration required.`
explicitly instead of leaving the migration section empty.

`CHANGELOG.md` remains the historical ledger. These docs are the guided
release surface for humans and agents adopting a release.
