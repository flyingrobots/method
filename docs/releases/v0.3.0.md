# v0.3.0

First public release of METHOD.

## Summary

METHOD is a calm development method for a repo shared by a human and
an agent. This release ships a working CLI, MCP server, GitHub
adapter, drift detection, and full documentation — 29 development
cycles and 127 tests.

## What Changed

- **CLI**: 9 commands covering the full METHOD loop — from `method
  init` to `method sync ship`.
- **MCP Server**: 8 tools for agent integration. Every tool takes an
  explicit `workspace` path, so one server can serve multiple projects.
- **GitHub Sync**: Two-way synchronization between local backlog and
  GitHub Issues. Push local changes, pull remote labels/comments/status.
- **Drift Detection**: Compares playback questions against test
  descriptions with near-miss hints for close-but-not-exact matches.
- **Configurable Paths**: `.method.json` lets projects customize where
  METHOD puts backlog, design, retro, and other directories.
- **Hybrid Signpost Generation**: Hand-authored prose stays; sections
  marked with `<!-- generate:NAME -->` are regenerated from source on
  every ship sync.
- **OSS Scaffolding**: Apache 2.0 license, CONTRIBUTING, SECURITY,
  NOTICE, ARCHITECTURE.

## Why It Matters

METHOD is designed for solo developers working with AI agents. It
replaces ad-hoc project management with filesystem-native backlog
discipline, small numbered cycles, and reproducible witnesses. The
MCP server means your agent can operate METHOD programmatically.

## Breaking Changes

None. This is the first release.

## Migration

No migration required.
