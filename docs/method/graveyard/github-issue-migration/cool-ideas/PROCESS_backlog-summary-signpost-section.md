# Backlog Summary Signpost Section

Add a generated backlog-summary section to a signpost such as
`docs/GUIDE.md` or another repo-facing doc. The section should provide a
bounded snapshot of lane counts, active pressure, and notable priority
clusters without dumping the full backlog or forcing agents to expand a
large MCP payload just to answer "where are we?".
