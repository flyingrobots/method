# Release Packets

Store internal release artifacts here.

Each shaped release should have a versioned directory such as
`docs/method/releases/vX.Y.Z/` containing:

- `release.md` for release design, included scope, sponsored-user
  impact, and version justification
- `verification.md` for pre-flight, publish, and delivery evidence

These are internal METHOD artifacts. User-facing release notes belong in
`docs/releases/`.
