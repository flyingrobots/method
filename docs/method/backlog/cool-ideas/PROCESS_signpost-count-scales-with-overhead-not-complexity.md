# Signpost count scales with overhead not complexity

8 signposts (BEARING, VISION, GUIDE, PROCESS, RELEASE, CLI, MCP, ARCHITECTURE) for ~2000 lines of logic. Each signpost is a maintenance surface. The overhead scales with signpost count, not project complexity. Consider: which signposts are load-bearing for all METHOD repos vs which are specific to this repo's maturity? Should new repos start with fewer signposts and grow into them?
