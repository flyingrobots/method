# Maintenance track for sub-cycle work

The maint-slug branch convention exists in process.md but has no tooling. Updating a dependency, fixing a doc typo, bumping a version — these don't need design docs, playback questions, or retros. Add a maintenance track: method maint or similar, which creates a branch, does the work, and commits without cycle machinery. This pairs with the t-shirt sizing idea — S-sized work should use the maintenance track.
